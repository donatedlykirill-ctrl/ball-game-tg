🗑️ ШАГ 1: УДАЛИТЬ СТАРЫЙ ПРОЕКТ (2 минуты)
────────────────────────────────────────────────────────────────

1. Открой https://vercel.com/dashboard

2. Найди проект "ball-game-backend"

3. Нажми на строку с проектом (она раскроется)

4. Нажми три точки (...) справа сверху

5. Нажми "Settings"

6. Скролли вниз (очень вниз!)

7. Найди кнопку "Delete Project"

8. Нажми её

9. Когда спросит подтверждение - нажми Delete

✅ Проект удален!

════════════════════════════════════════════════════════════════

🆕 ШАГ 2: СОЗДАТЬ НОВЫЙ ПРОЕКТ (10 минут)
────────────────────────────────────────────────────────────────

1. На Vercel Dashboard нажми "Add New..."

2. Выбери "Project"

3. Нажми "Import Git Repository"

4. В поле вставь:
   https://github.com/donatedlykirill-ctrl/ball-game-tg

5. Нажми Import

6. БУДЕТ ЭКРАН КОНФИГУРАЦИИ - ЗДЕСЬ ВАЖНО!

   Ищи поле "Framework Preset:"
   Нажми на выпадающее меню
   Выбери: *** Node.js ***
   
   Ищи поле "Root Directory:"
   Нажми "Edit"
   Найди папку "backend"
   Выбери её
   Нажми "Save"
   
   Поле "Build Command:" должно быть ПУСТО
   Если там что-то написано - очисти!
   
   Поле "Output Directory:" должно быть ПУСТО
   Если там что-то написано - очисти!

7. Скролли вниз до "Environment Variables"

8. Нажми "Add New" (или + кнопка)

9. Добавь эти переменные по одной:

   1️⃣ Name: MYSQL_HOST
      Value: 127.0.0.1
      Нажми "Add"

   2️⃣ Name: MYSQL_USER
      Value: aruvaworkg
      Нажми "Add"

   3️⃣ Name: MYSQL_PASSWORD
      Value: jCc-Urj-Sba-G3i
      Нажми "Add"

   4️⃣ Name: MYSQL_DATABASE
      Value: aruvaworkg
      Нажми "Add"

   5️⃣ Name: MYSQL_PORT
      Value: 3308
      Нажми "Add"

   6️⃣ Name: NODE_ENV
      Value: production
      Нажми "Add"

   7️⃣ Name: PORT
      Value: 3000
      Нажми "Add"

10. Нажми кнопку "Deploy" (она красная)

11. ЖДИ 3-5 МИНУТ!

    Будет видна полоса прогресса
    Обычно пишет:
    - "Installing dependencies..."
    - "Building..."
    - "Deploying..."

❌ ЕСЛИ ОШИБКА...
────────────────────────────────────────────────────────────────

Видишь красную ошибку?

1. Нажми на проект

2. Посмотри Deployments (история)

3. Нажми на красный deployment

4. Вкладка "Build Logs"

5. Читай что там написано

Типичные ошибки:
  - "Cannot find module" → нужны зависимости
  - Port error → проверь PORT = 3000
  - MySQL error → проверь credentials
