❓ ВОПРОС: Что выбирать в "Application Preset" или "Framework Preset"?

✅ ОТВЕТ: Node.js

════════════════════════════════════════════════════════════════

📸 ШАГ ЗА ШАГОМ
────────────────────────────────────────────────────────────────

1. На Vercel.com нажимаешь "Add New..." → "Project"

2. "Import Git Repository" → выбираешь ball-game-tg

3. Попадаешь на экран конфигурации

   На этом экране видишь выпадающее меню:
   
   ┌─────────────────────────────────────────┐
   │ Framework Preset                    ▼   │
   │ ┌─────────────────────────────────────┐ │
   │ │ Node.js              ← ВЫБЕРИ ЭТО! │ │
   │ │ Next.js                             │ │
   │ │ Nuxt                                │ │
   │ │ Vite                                │ │
   │ │ Other                               │ │
   │ └─────────────────────────────────────┘ │
   └─────────────────────────────────────────┘

   Нажимаешь "Node.js"

☐ Жди 2-3 минуты

❓ ЧТО ЕСЛИ НЕ ВИЖУ "Framework Preset"?
────────────────────────────────────────────────────────────────

Это нормально! Vercel может автоматически обнаружить Node.js

Проверь:
✓ Root Directory установлен на "backend"
✓ В папке backend есть package.json
✓ В package.json есть "main" или "scripts.start"

Если всё есть - просто переходи к Environment Variables
