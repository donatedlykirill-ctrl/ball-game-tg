🚀 NEXT STEPS (в порядке выполнения)
──────────────────────────────────────────────────────────────────

STEP 1️⃣: GitHub (5 минут)
  1. Открой: https://github.com/new
  2. Создай репозиторий:
     - Name: ball-game-tg
     - Public
     - Add .gitignore (Node)
  3. Скопируй URL: https://github.com/YOUR_USER/ball-game-tg.git
  4. Выполни в PowerShell:
  
     git remote add origin https://github.com/YOUR_USER/ball-game-tg.git
     git push -u origin main
  
  5. При запросе пароля:
     - Открой: https://github.com/settings/tokens
     - New token (classic)
     - ☑️ repo scope
     - Generate → скопируй и вставь

STEP 2️⃣: Backend на Vercel (3 минуты)
  1. Открой: https://vercel.com
  2. Sign Up → Continue with GitHub
  3. Import Project → GitHub → ball-game-tg
  4. Конфигурация:
     - Framework: Node.js
     - Root Directory: backend
     - Build Command: (оставь пустым)
     - Environment Variables (добавь):
       MYSQL_HOST = 127.0.0.1
       MYSQL_USER = aruvaworkg
       MYSQL_PASSWORD = jCc-Urj-Sba-G3i
       MYSQL_DATABASE = aruvaworkg
       MYSQL_PORT = 3308
       NODE_ENV = production
  5. Deploy! (ждешь 2-3 минуты)
  6. СКОПИРУЙ ПОЛУЧЕННЫЙ URL!
     Выглядит: https://ball-game-backend-xxx.vercel.app

STEP 3️⃣: Frontend на Vercel (3 минуты)
  1. На Dashboard: Add New → Project
  2. Import Git Repository
  3. Выбери: ball-game-tg
  4. Конфигурация:
     - Framework: Vite
     - Root Directory: .
     - Build Command: npm run build
     - Output Directory: dist
     - Environment Variables:
       VITE_APP_API_URL = https://ball-game-backend-xxx.vercel.app
       
       ⚠️ ЗАМЕНИ xxx на ID из backend URL!
  5. Deploy!
  6. ПОЛУЧИШЬ ФИНАЛЬНЫЙ URL!
     Выглядит: https://ball-game-frontend-xxx.vercel.app

STEP 4️⃣: Проверка (1 минута)
  1. Открой frontend URL в браузере
  2. Нажми Play
  3. Запусти игру (Launch Blue Ball)
  4. Пусти раунд до конца
  5. Проверь консоль (F12 → Console) что данные сохраняются
