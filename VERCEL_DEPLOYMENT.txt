📋 ЭТАП 2: BACKEND НА VERCEL (3 минуты)

1️⃣ Открой Vercel Dashboard:
   https://vercel.com/dashboard

2️⃣ Нажми "Add New..." → "Project"

3️⃣ Нажми "Import Git Repository"

4️⃣ В поле вставь GitHub URL:
   https://github.com/donatedlykirill-ctrl/ball-game-tg

5️⃣ После выбора репозитория настрой:

   ✓ Framework Preset:
     → Node.js

   ✓ Root Directory:
     → Нажми "Edit"
     → Выбери "backend"
     → Нажми "Save"

   ✓ Build Command:
     → Оставь пустым (по умолчанию: npm run dev)

   ✓ Environment Variables:
     → Нажми "Add +" и добавь каждую переменную:
     
     MYSQL_HOST
     Value: 127.0.0.1
     
     MYSQL_USER
     Value: aruvaworkg
     
     MYSQL_PASSWORD
     Value: jCc-Urj-Sba-G3i
     
     MYSQL_DATABASE
     Value: aruvaworkg
     
     MYSQL_PORT
     Value: 3308
     
     NODE_ENV
     Value: production

6️⃣ Нажми "Deploy"

7️⃣ ⏳ Жди 2-3 минуты (наблюдай за строкой статуса)

📋 ЭТАП 3: FRONTEND НА VERCEL (3 минуты)
────────────────────────────────────────────────────────────────

1️⃣ На Vercel Dashboard нажми "Add New..." → "Project"

2️⃣ Нажми "Import Git Repository"

   ✓ Output Directory:

6️⃣ ⏳ Жди 2-3 минуты

❌ ЕСЛИ ЕСТЬ ОШИБКИ
────────────────────────────────────────────────────────────────

Ошибка: "Failed to load player"
→ Проверь VITE_APP_API_URL в frontend Environment Variables
→ Убедись что backend URL правильный

Ошибка: "Cannot connect to database"
→ Проверь MySQL credentials в backend Environment Variables
→ Убедись что MYSQL_HOST и PORT правильные

Ошибка: "Build failed"
→ Посмотри Deployment Log на странице проекта
→ Проверь что package.json правильный
